HTML Interview Question and Answers

HTML (HyperText Markup Language) is the standard language used to create and design web pages. It structures web content by using a system of elements or tags that define the different parts of a webpage, such as text, images, links, and multimedia.

HTML creates the basic structure for a website, CSS adds style to that structure, and JavaScript makes it interactive and more functionally complex.

Doctype is the first line of an HTML document which tells the version of HTML that we are using.

Tags are used to define the beginning and end of an element, while Elements consist of the opening tag, content, and closing tag.

HTML entities are reserved characters that have special meanings when used in an HTML document. Each of these codes starts with an ampersand and ends with a semicolon.

HTML 5 improves upon HTML 4 by adding more features like multimedia support, a canvas for drawing, storage for web apps to work offline, and a way to find a user's location.

Start tag: Marks the beginning of the element's content. For example, the start tag for a paragraph is <p>
Content: The content of the element, which can be text or other elements.
End tag: Marks the end of the element's content. The end tag is the same as the start tag, but with a forward slash (/) after the first angle bracket. For example, the end tag for a paragraph is </p>.

We can use a <div> tag with width and height for making an empty square box without any content.

Attr refers to attributes in HTML. An attribute provides additional information about a particular tag. Some common attributes are:
  • Source attribute (src)
  • Hyperlink reference attribute (href)
  • Alternate (alt)
  • Target
  • Language (lang)

  • Title
  • Style
  • Meta
  • Link
  • Script

Meta tags are used in HTML at the head of the page, making them visible only to search engines and those who know how to view website source code. Metadata helps search engines understand the data on a website, aiding in content ranking on popular search engines like Google.

SEO stands for 'Search Engine Optimization', which is the process of getting traffic from free, organic, editorial, or natural search results in search engines. It aims to improve a website's position in search results pages (SERPs). Higher listings result in more visibility.

An <iframe> is a tag that allows a browser to display another document within the current page. Attributes like src, height, width, and others define and modify the region created by the <iframe> tag.

A form is used to collect detailed information. Form elements include:
  • Input
  • Label
  • Select
  • Legend
  • Fieldset
  • Option
  • Textarea

Semantic elements provide detailed information about the content. Examples include <article>, <header>, <footer>.
Non-semantic elements do not provide information about the content. Examples include <span>, <div>.

Relative file paths reference files located in the same folder of your website structure, while absolute file paths refer to external resources located elsewhere on the web.

In HTML, the "src" attribute specifies the source of an embedded resource like an image, while the "href" attribute defines the destination URL of a hyperlink, indicating where a link will take the user when clicked. "Src" is used to display content within a page, while "href" is used to navigate to a different page.

Some inline elements include:
  • Bold
  • Span
  • Strong
  • Italic
  • Subscript
  • Superscript
  • Image
  • Insert
  • Emphasis

Git & GitHub Interview Question and Answers

Git is a version control system that tracks changes in computer files. It is used to push the code from our system.

Git is an open-source version control tool that tracks changes to files, while GitHub is a cloud-based platform that allows users to share and collaborate on code and helps to do projects together.

The git init command is used to initialize Git in your local directory.

  • Working directory
  • Staging area
  • Local repository
  • Remote repository

git add . is used to add all files and git add is used to add a particular file.

git commit -m "message"

git remote add origin

A repository is a place where you can store your code, your files, and each file's revision history.

git push origin main

git branch -M branchname

git branch

git checkout

git log --oneline

The git status command shows the current state of your Git repository, including the status of your working directory and staging area.

git fetch, git pull, git clone

A fork in GitHub is a copy of a repository that shares the same code and visibility settings as the original repository.
  1. Find the repository: Go to the repository page that you want to fork.
  2. Fork the repository:
    • In the top-right corner of the repository's page, you'll see a "Fork" button.
    • Click on the "Fork" button. GitHub will create a copy of the repository under your account.

  • git fetch: Downloads commits, files, and references from a remote repository, but doesn't merge or modify your working directory.
  • git pull: Combines git fetch and git merge—it fetches the latest changes and directly updates your local branch by merging them.

  1. git stash:
    • Temporarily saves (stashes) your uncommitted changes without committing them, allowing you to work on something else.
    • You can later reapply these changes using git stash pop or git stash apply.
  2. git diff:
    • Shows the differences between your working directory and the last commit (or between any two commits).
    • Useful for seeing changes made to files before committing them.

  1. git remote:
    • Manages connections to remote repositories (e.g., GitHub).
    • You can use it to add, remove, or list remotes.
  2. git reset:
    • Moves the HEAD pointer and optionally modifies the working directory and staging area.
    • Useful for undoing commits or unstaging changes.

CSS Interview Question and Answers

CSS is the acronym of “Cascading Style Sheets”. CSS is a computer language for laying out and structuring web pages (HTML or XML).

A CSS Syntax rule consists of a selector, property, and its value.
selector{
property:value;
}

CSS can be added to HTML documents in 3 ways:
Inline - by using the style attribute inside HTML elements.
Internal - by using a < style > element in the < head > section.
External-To link the CSS to an HTML file, we use the < link > tag inside the HTML < head > section.

There are several different types of selectors in CSS.
1. CSS Element Selector.
2. CSS Id Selector.
3. CSS Class Selector.
4. CSS Universal Selector.
5.CSS Group Selector.

A Class name can be used by multiple HTML elements, while an ID name must only be used by one HTML element within the page. Each of these codes starts with an ampersand and ends with a semicolon.

Colors in CSS can be specified by the following methods:
1.Hexadecimal colors. (e.g., #ff0000, #00ff00)
2.RGB colors.RGB values (e.g., rgb(255, 0, 0), rgb(0, 255, 0))
3.Predefined/Cross-browser Color names (e.g., red, blue)

with display: inline-block , the top and bottom margins/paddings are respected, but with display: inline they are not. Compared to display: block , the major difference is that display: inline-block does not add a line-break after the element, so the element can sit next to other elements.

A span is an inline element used to group and style text or inline elements, while a div is a block-level element used to create sections or divisions in a document and style them as a whole.

Float allows other HTML elements to flow around the floating element. Align simply shifts an element to the left or right. Align - You use align to align text and other items rather it be left, right, centered, or justified. Align DOES NOT remove the item from the document flow.

Pseudo-classes enable you to target an element when it's in a particular state, as if you had added a class for that state to the DOM.Pseudo-classes target states like ': hover' or ': active',
Pseudo-elements act as if you had added a whole new element to the DOM, and enable you to style that.Pseudo-elements like ' ::before ' or:: after' style specific parts of an element.

The CSS box model module defines the height , width , margin , and padding properties, which along with border properties make up the CSS box model.

In CSS, a margin is the space around an element's border, while padding is the space between an element's border and the element's content.

Relative - the element is positioned relative to its normal position.
Absolute - the element is positioned absolutely to its first positioned parent.

The position: fixed mean fixed to the viewport. We tell it where to position itself (top, bottom, right, or left) and it will stay there when user scrolling.
Meanwhile, when using position: sticky it won't affect until we define offset, like top: 10px.

Position property determines in what manner an item is positioned on the page or relative to one another.
CSS's Display property specifies the type of rendering box that is created from our HTML you can say it like block, inline, inline block, flex.

Transforms move or change the appearance of an element,
Transitions make the element smoothly and gradually change from one state to another.

The transition-timing-function property specifies the speed curve of the transition effect. This property allows a transition effect to change speed over its duration.
The transition-duration CSS property sets the length of time a transition animation should take to complete. By default, the value is 0s , meaning that no animation will occur.

Grid and flexbox.
flexbox was designed for layout in one dimension - either a row or a column.
Grid was designed for two-dimensional layout - rows, and columns at the same time.

CSS media queries are a feature in CSS that allows you to apply styles to an element based on the characteristics of the device or screen displaying the content. This includes factors like screen width, height, resolution, orientation, and more.

Responsive web design. Adaptive web design. Dynamically changes a site's page layout based on a device's display type, width, height, etc. Detects the screen size and loads the appropriate layout for it. Uses CSS media queries to change styles.

Z-index is the CSS property that controls the stacking order of overlapping elements on a page. An element with a higher z-index value will appear in front of an element with a lower z-index value. The property is called “z-index” because it sets the order of elements along the z-axis.
